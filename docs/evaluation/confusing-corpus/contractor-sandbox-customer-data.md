# Contractor Sandbox Customer Data Handling Standard

## Purpose

This standard controls contractor use of customer-like data in sandbox, staging, test, and development environments.

## Policy

Contractors may use customer data in a sandbox only when the data has been masked, tokenized, or synthetically generated before it enters the sandbox. Production customer data must not be copied directly into sandbox environments.

Sandbox access for contractors requires project sponsor approval and security review approval. The approval record must include contractor identity, sandbox environment, masking method, data category, and expiry date.

Sandbox access may remain active for up to thirty days and must be removed when the project ends. Emergency production access rules do not authorize sandbox copying.

## Evidence

Required evidence includes masking validation, project sponsor approval, security review approval, sandbox access grant, and removal confirmation.
