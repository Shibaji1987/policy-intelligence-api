# Employee Break Glass Production Access Standard

## Purpose

This standard controls employee break glass access to production systems during severe incidents.

## Policy

Employees may use break glass production access only when normal privileged access is unavailable and an active severe incident is declared. Break glass access requires incident commander approval and must expire within two hours.

Break glass access may expose production customer data incidentally, but employees must limit activity to incident containment and service restoration. Break glass access does not authorize contractor access and does not authorize customer data export.

Every break glass session must be reviewed by security within one business day.

## Evidence

Required evidence includes incident declaration, incident commander approval, break glass session log, security review, and closure note.
